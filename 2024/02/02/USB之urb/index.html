USB之urb
2024-02-02|

在USB设备驱动程序中，"URB buffer"和"URB"是两个密切相关但又有所区别的概念，它们共同参与USB数据传输的处理。

URB（USB Request Block）

  • 定义：URB是USB请求块的缩写，用于Linux内核的USB子系统中。每个URB描述了一个USB事务的全部或部分，包括了所有需要的信息来执行这个请求，如目标USB设备、端点、数据方向（输入或输出）、传输类型（如控制、批量、中断或同步）等。
  • 作用：URB是USB驱动和USB核心之间交互的基本单位。驱动程序通过填充并提交URB给USB核心来请求数据传输，USB核心根据URB描述的细节执行具体的数据传输操作。完成后，USB核心会通知驱动程序传输的结果。

创建和使用URB

  1. 分配URB：使用usb_alloc_urb函数分配一个新的URB。这个函数的参数指定了传输使用的ISO包的数量（对于非ISO传输，此参数为0）。
  2. 设置URB参数：填充URB结构的字段，包括USB设备指针、端点信息、数据缓冲区、数据长度、完成处理函数等。
  3. 提交URB：通过调用usb_submit_urb函数，将配置好的URB提交给USB核心。这个函数返回后，URB会被USB核心处理，数据传输在后台进行。
  4. 处理完成：当URB完成时，之前指定的完成处理函数将被调用。这个函数可以检查URB的状态，处理传输的数据，并决定是否重新提交URB。
  5. 释放URB：使用usb_free_urb函数释放URB所使用的资源。通常在URB不再需要时调用。
