多路I/O复用poll、select、epoll
2024-03-31|

什么是IO多路复用

I/O多路复用是一种允许程序同时等待多个I/O操作的技术，它可以让单个进程或线程同时监视多个文件描述符以检测它们是否有I/O操作可执行（例如，数据可读、可写或出现异常等）。当某个或某些文件描述符准备好进行I/O操作时，程序就会被唤醒来处理相应的事件。

常用的I/O多路复用方法

  1. select

    • select是最基本的多路I/O复用实现，几乎所有支持POSIX的操作系统都支持select。它允许应用程序监视一组文件描述符，等待其中一个或多个成为"就绪"状态。
    • 缺点：它有一些限制，包括固定的最大文件描述符限制（FD_SETSIZE，通常为1024）和每次调用时都需要重新构建文件描述符集和线性扫描，这在文件描述符数量较大时效率较低。

  2. poll

    • poll与select类似，但没有文件描述符数量的限制。poll使用一个pollfd结构数组来存储需要监视的文件描述符及其请求的事件和发生的事件。
    • 缺点：与select一样，随着需要监控的文件描述符数量增加，性能会线性下降。

  3. epoll（仅限Linux）

    • epoll是Linux特有的多路I/O复用机制，相比于select和poll，epoll在处理大量文件描述符时更加高效。epoll可以在O(1)的时间复杂度下操作文件描述符，而不是O(n)。
    • epoll有两种工作模式：LT（水平触发）和ET（边缘触发）。ET模式允许更高效的事件处理。
