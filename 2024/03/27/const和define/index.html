const和define
2024-03-27|

const 关键字和宏定义（#define）都可用于定义不变的值，但有一些区别

const 关键字

  • 类型安全：const提供了类型安全，编译器会检查与该类型相关的操作是否合法。
  • 存储：const 变量通常存储在程序的数据段中。它是一个具有确切内存位置的变量，这意味着可以使用指针对其进行操作。

#define 宏定义

  • 预处理器：#define 是一个预处理指令，用于定义宏。它在编译之前由预处理器执行，用于文本替换。
  • 类型不安全：#define 没有类型，可以用于定义字面值、表达式、代码片段等。因此，它不提供类型安全。
  • 存储：由于 #define 是在预处理阶段进行文本替换，所以它不占用存储空间。

const 和 #define 的主要区别

  1. 类型安全：const 是类型安全的，而 #define 不是。使用 const 可以让编译器提供类型检查，减少错误。
  2. 作用域控制：const 变量有明确的作用域，而 #define 定义的宏则在其定义之后的整个编译单元内都是可见的。
  3. 内存分配：const 为其变量分配内存，而 #define 只是简单地在预处理阶段进行文本替换，不分配内存。
